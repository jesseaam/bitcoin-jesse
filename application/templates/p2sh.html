Pay to Script Hash (P2SH)

Overview

Pay to Script Hash (P2SH) is a method for simplifying multisig transactions, especially for the payer. Recall, the format of the ScriptPupKey (locking script) for a multisig tx is:

                    
2 "Public Key1" "Public Key2" "Public Key3" "Public Key4" "Public Key5" 5 CHECKMULTISIG

Let's break this down. In general, a multisig transaction requires M of N signatures to spend the bitcoin. The format of the locking script is:

  1. The quorum
  2. List of each possible public key
  3. The total number of public keys
  4. OP_CHECKMULTISIG.

So here we have the quorum=2. This means that the payee must provide 2 valid signatures in the ScriptSig (unlocking script) order to spend the bitcoin. Next we have public keys 1â€“5. The payee must provide valid signatures for 2 of these 5 public keys. The following number 5 tells us how many public keys there are in total. Lastly is the opcode CHECKMULTISIG. Note that in the ScriptSig, signatures must be in the same order as their corresponding public key. Check our the Bitcoin Wiki Script Page for information on the algorithm that is used.
